Pallet Bridge

A permissionless composability bridge that enables any Solidity contract on Polkadot Asset Hub to call any Substrate pallet dispatchable. No SCALE on-chain. Runtime-upgrade safe. Public good.

Usage from Solidity

Call any pallet through the bridge by passing SCALE-encoded call data:

AssetSwap.sol
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

contract AssetSwap {
    address public constant RUNTIME_PALLETS =
        0x3951D3C715247994d47D9382969cd0082967b258;

    event PalletDispatched(address indexed caller, bool success, uint256 dataLength);

    function dispatchPallet(bytes calldata scaleCallData)
        external returns (bool success)
    {
        require(scaleCallData.length > 0, "empty call");
        (success, ) = RUNTIME_PALLETS.call(scaleCallData);
        emit PalletDispatched(msg.sender, success, scaleCallData.length);
    }

    function dispatchPalletOrRevert(bytes calldata scaleCallData) external {
        (bool success, ) = RUNTIME_PALLETS.call(scaleCallData);
        require(success, "pallet dispatch failed");
        emit PalletDispatched(msg.sender, true, scaleCallData.length);
    }
}

Or call RUNTIME_PALLETS_ADDR directly without the bridge contract:

Direct Call
address constant RUNTIME_PALLETS = 0x3951...b258;

function directPalletCall(bytes calldata scaleData) external {
    (bool success, ) = RUNTIME_PALLETS.call(scaleData);
    require(success, "pallet dispatch failed");
}

Usage from Frontend (PAPI + viem)

The frontend encodes the pallet call using PAPI (which has the full runtime metadata), then submits it through a Revive.call extrinsic.

TypeScript / PAPI
import { createClient, Binary } from 'polkadot-api'
import { getWsProvider } from 'polkadot-api/ws'
import { paseo_ah } from '@polkadot-api/descriptors'

const client = createClient(getWsProvider('wss://asset-hub-paseo.dotters.network/'))
const api = client.getTypedApi(paseo_ah)

// Step 1: Encode pallet call as SCALE bytes
const swapCall = api.tx.AssetConversion.swap_exact_tokens_for_tokens({
  path: [
    { parents: 1, interior: { Here: undefined } },
    { parents: 0, interior: { X2: [
      { PalletInstance: 50 },
      { GeneralIndex: 1984n }
    ]}}
  ],
  amount_in: 1_000_000_000_000_000_000n,
  amount_out_min: 0n,
  send_to: '5GrwvaEF...',
  keep_alive: true,
})

const scaleBytes = swapCall.getEncodedData()

// Step 2: Submit via Revive.call extrinsic
const tx = api.tx.Revive.call({
  dest: Binary.fromHex('bb5a441bfce51c5b1fd5cd603b87c9787a8ba7b7'),
  value: 0n,
  gas_limit: { ref_time: 500_000_000_000n, proof_size: 500_000n },
  storage_deposit_limit: 1_000_000_000_000n,
  data: Binary.fromHex(evmCallData),
})

await tx.signAndSubmit(signer)

How RUNTIME_PALLETS_ADDR Is Computed

The address is derived deterministically and is identical across all pallet-revive deployments:

Address Derivation
# PalletId = *b"py/paddr"
# blake2b_256(b"modl" ++ b"py/paddr" ++ [0u8; 24])[:20]
# = 0x3951D3C715247994d47D9382969cd0082967b258

# Verify in Python:
from hashlib import blake2b

preimage = b"modl" + b"py/paddr" + bytes(24)
digest = blake2b(preimage, digest_size=32).digest()
evm_address = "0x" + digest[:20].hex()
# => 0x3951d3c715247994d47d9382969cd0082967b258

Security Considerations

  • Origin is the contract, not the user. Pallet calls execute with the contract's Substrate origin, not the end user's.
  • Permissionless dispatch. No access control on which pallets can be called. This is by design — access control belongs in the calling contract.
  • No SCALE stored on-chain. Call data is passed through as calldata. Runtime upgrades that change pallet indices do not break the bridge.
  • Call data validation. The bridge checks only that the payload is non-empty. Malformed SCALE bytes are rejected by the runtime.
  • Value transfer. The bridge does not forward msg.value. The contract's mapped account must be funded separately.

Deploy on Mainnet

The bridge is deployed on Paseo Asset Hub (testnet). Polkadot Asset Hub deployment is planned once pallet-revive is live on mainnet. The contract is permissionless — anyone can deploy an instance.
